# 安全边界与 MVP 路线图

本文用于理解执行过程中的安全限制、第一阶段 MVP 范围，以及后续阶段的演进方向。

## 如何使用本文

- 当任务只涉及本文主题时，优先只加载本文作为上下文。
- 本文保留原始设计文档中的关键实现要求，并补充少量阅读说明。
- 本次拆分只改变文档组织方式，不改变原有系统架构结论。

## 13\. 安全与执行边界

## 13.1 必须控制的风险

- 不应默认允许删除、支付、发布等高风险操作
- 不应让 agent 自由执行 arbitrary code
- 不应在真实生产环境直接跑高危 persona
- 不应使用真实用户账号与真实敏感数据

---

## 13.2 安全措施

- 仅在测试环境运行
- 引入 allowlist 动作集合
- 引入页面级风险标记
- 对 destructive action 设置二次确认或禁止
- 对外部网络、支付、删除接口做环境隔离
- 敏感数据脱敏

---

## 15\. MVP 范围

## 15.1 第一阶段建议实现

### 功能

- 支持 3 类 persona
- 支持 3 类任务模板
- Playwright 执行网页任务
- 保存 step 日志和截图
- 基础 RAG：产品知识 + 历史失败
- 基础摩擦报告
- 基础前端 run 查看页

### 技术

- FastAPI
- LangGraph
- Playwright
- PostgreSQL
- 本地文件存储截图
- 可选 Redis（先不用也行）

---

## 15.2 第二阶段

- UI 经验库
- 版本对比
- 实时事件流
- Redis + worker
- 前端步骤回放
- 更细粒度摩擦分类

---

## 15.3 第三阶段

- 多 persona 并发测试
- AI 产品专项 rubric
- 更复杂恢复分支
- 桌面/移动端扩展
- 视觉 grounding 能力增强
