# 收敛运行报告结论并细化问题报告

## Commit

收敛运行报告结论并细化问题报告

## 变更目标

让当前最小 run 闭环输出更接近真实项目目标的最终报告：不仅要给出成功或失败摘要，还要统一输出可消费的结论级别、说明问题出现在哪些步骤、为什么当前 persona 更容易在这些位置失败或犹豫，并让异常中断路径也保持同样的报告结构。

## 修改前

`RunReport` 还没有统一的结论字段，报告生成逻辑主要停留在“简短摘要 + 静态建议”，对本次 run 的真实步骤、摩擦信号、最后一步判断和 persona 行为特征利用不足；异常补报告路径也没有显式写入同一套结论级别，导致正常收尾与异常收尾的报告结构存在差异。

## 修改后

当前 `RunReport` 已增加 `conclusion` 字段，且只允许输出 `keep` / `optimize` / `fix`；`report_builder` 会先按真实执行结果推导基础结论，再结合 steps、摩擦信号、最后一步 `progress_summary`、persona 与 task 生成更详细的 `key_findings`，并在需要时补充只针对本次 run 问题的 `next_recommendations`。`backend/graph/run_graph.py` 的异常补报告分支也固定写入 `conclusion="fix"`，确保正常收尾与异常收尾路径的报告结构一致。

## 修改文件

- `backend/schemas/run_schemas.py`
  - 修改原因：需要为最终报告补充统一、可验证的结论字段。
  - 修改内容：为 `RunReport` 新增 `conclusion` 字段，并定义 `ReportConclusion` 只允许 `keep` / `optimize` / `fix` 三档英文值。
  - 修改影响：后续所有报告生成、异常补报告和测试断言都需要遵守同一结论枚举。

- `backend/analysis/report_builder.py`
  - 修改原因：需要把最终报告从简短摘要收敛为更贴近真实项目目标的详细问题报告。
