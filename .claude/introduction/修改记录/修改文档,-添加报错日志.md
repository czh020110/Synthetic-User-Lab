# 修改文档,-添加报错日志

## Commit

修改文档, 添加报错日志

## 变更目标

补齐最小 Run 闭环的失败可观测性，让异常状态、失败报告和失败原因能够在查询链路中保留。

## 修改前

run 初始化或执行失败时，状态信息和报告链路对失败原因的保留不完整，排查问题需要额外看日志，`/report` 也可能长期返回未完成状态。

## 修改后

异常失败会保留更明确的失败信息，并在无最终报告时生成最小失败报告；同时补充浏览器会话创建与异常分支日志，提升排查效率。

## 修改文件

- `backend/graph/demo_run_graph.py`
  - 修改原因：异常失败链路可观测性不足。
  - 修改内容：在 run 异常分支中保留更明确的失败信息，并在必要时补齐最小失败报告。
  - 修改影响：状态接口和报告接口可以返回更可读的失败结果。
- `backend/stores/in_memory_run_store.py`
  - 修改原因：失败状态写回后错误信息会被清空。
  - 修改内容：调整 complete_run 的错误信息处理，失败时保留 error_message。
  - 修改影响：失败状态排查不再丢失根因。
- `backend/execution/playwright_adapter.py`
  - 修改原因：浏览器初始化失败难以定位。
  - 修改内容：补充会话创建日志与异常栈日志。
  - 修改影响：初始化问题可直接从服务日志定位。
- `introduction/数据流/核心数据流.md`
  - 修改原因：补充异常失败后的真实数据流。
  - 修改内容：更新失败分支、报告分支与查询接口之间的关系说明。
  - 修改影响：文档能反映失败后仍可查询报告的实际链路。
- `introduction/TODO/DONE.md`
  - 修改原因：同步已完成事项。
  - 修改内容：记录失败日志与失败报告链路已补齐。
  - 修改影响：进度文档与代码状态保持一致。
