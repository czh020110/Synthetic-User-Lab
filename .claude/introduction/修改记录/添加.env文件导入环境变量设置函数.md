# 添加.env文件导入环境变量设置函数

## Commit

添加.env文件导入环境变量设置函数

## 变更目标

让项目可以从根目录 `.env` 读取运行配置，降低本地启动与配置管理成本。

## 修改前

运行配置主要依赖代码默认值和系统环境变量，本地项目缺少统一配置入口，API 前缀等参数也不便集中维护。

## 修改后

配置模块支持从项目根目录 `.env` 加载环境变量，并把应用名、API 前缀、运行步数限制等参数纳入统一配置对象管理。

## 修改文件

- `backend/core/config.py`
  - 修改原因：缺少项目级 `.env` 配置加载能力。
  - 修改内容：新增 `.env` 读取逻辑，并将关键运行参数接入 `Settings`。
  - 修改影响：本地运行参数可通过 `.env` 统一维护，系统环境变量仍可覆盖。
- `backend/graph/demo_run_graph.py`
  - 修改原因：图运行依赖统一配置。
  - 修改内容：接入新的配置读取结果以复用运行时参数。
  - 修改影响：图入口与配置层保持一致。
- `introduction/环境说明/本地开发环境.md`、`introduction/环境说明/常见命令.md`
  - 修改原因：同步环境与命令事实。
  - 修改内容：补充 `.env`、环境选择与相关运行方式说明。
  - 修改影响：本地开发文档与配置能力对齐。
- `introduction/TODO/DONE.md`
  - 修改原因：同步已完成事项。
  - 修改内容：记录 `.env` 配置导入与 API 前缀规范化能力完成。
  - 修改影响：阶段进度可追踪。
