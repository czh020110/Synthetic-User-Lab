# RAG 记忆与上下文系统

本文用于理解本项目中的 RAG 不是普通问答，而是任务执行时的产品知识、UI 经验、失败恢复和评测规则供给层。

## 如何使用本文

- 当任务只涉及本文主题时，优先只加载本文作为上下文。
- 本文保留原始设计文档中的关键实现要求，并补充少量阅读说明。
- 本次拆分只改变文档组织方式，不改变原有系统架构结论。

## 6\. RAG 设计

## 6.1 RAG 在本项目中的定位

RAG 不是主任务，而是 **上下文供给与经验记忆层** 。

它用于帮助 agent 在决策前知道：

- 当前页面代表什么
- 该功能应该如何使用
- 类似场景以前如何成功
- 遇到类似失败时如何恢复
- 什么才算任务完成或严重摩擦

---

## 6.2 RAG 子系统划分

## 6.2.1 产品知识库（Product Knowledge RAG）

数据来源：

- 帮助文档
- FAQ
- onboarding 文案
- 功能说明
- PRD 片段
- Release Notes

用途：

- 理解产品术语
- 理解页面功能
- 判断推荐流程
- 判断权限/限制

---

## 6.2.2 UI 经验库（UI Experience RAG）

数据来源：

- 历史页面摘要
- 成功路径摘要
- 失败路径摘要
- 页面布局描述
- 摩擦点评注

用途：

- 识别相似页面
- 回忆高成功率路径
- 提高 locator 漂移下的鲁棒性
- 辅助判断主路径按钮

---

## 6.2.3 失败恢复库（Failure Memory RAG）

数据来源：

- 历史报错案例
- 恢复成功案例
- 死循环案例
- 误操作摘要

用途：

- 遇错时查相似失败
- 推荐恢复动作
- 避免重复尝试无效动作

---

## 6.2.4 评测规则库（Evaluation Rules RAG）

数据来源：

- 任务成功 rubric
- UX heuristics
- 安全规范
- 可访问性要求
- 团队 QA checklist

用途：

- 判断是否成功
- 判断是否高摩擦
- 判断是否严重缺陷

---

## 6.2.5 Persona 行为先验库（可选）

数据来源：

- 各类用户行为模式
- 新手/专家典型路径
- 常见误解模式

用途：

- 让 persona 不只是 prompt 标签，而是有可检索行为偏好

---

## 6.3 检索策略

建议采用：

- **向量检索** ：找相似页面、相似案例
- **关键词检索 / BM25** ：找报错词、术语、按钮文案
- **Metadata Filter** ：按版本、页面类型、任务类型、persona 类型过滤

不建议只做单一向量检索。
