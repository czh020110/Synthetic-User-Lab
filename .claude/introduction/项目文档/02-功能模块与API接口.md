# 功能模块与 API 接口

本文用于理解系统需要实现哪些核心模块，以及这些模块对外暴露的基础 API 形态。

## 如何使用本文

- 当任务只涉及本文主题时，优先只加载本文作为上下文。
- 本文保留原始设计文档中的关键实现要求，并补充少量阅读说明。
- 本次拆分只改变文档组织方式，不改变原有系统架构结论。

## 5\. 系统功能设计

## 5.1 用户画像系统（Persona System）

## 功能

构建多种 synthetic user 画像，用于模拟不同群体行为。

## Persona 维度

- 用户水平：新手 / 熟练 / 专家
- 耐心程度：低 / 中 / 高
- 风险偏好：保守 / 激进
- 目标明确度：清晰 / 模糊
- 恢复能力：强 / 弱
- 术语理解能力：强 / 弱
- 对 AI 信任程度：高 / 低

## 输出

结构化 persona 配置，例如：

- 人设摘要
- 行为偏好
- 常见误操作倾向
- 恢复策略倾向
- 对引导文案敏感度

---

## 5.2 任务编排系统（Task System）

## 功能

将自然语言测试目标编译为可执行任务。

## 任务字段

- 任务名称
- 起始 URL
- 前置条件
- 成功条件
- 最大步数
- 最大错误次数
- 允许动作集合
- 风险等级
- 是否允许 destructive action

## 示例

任务：“注册后上传 PDF 并获取摘要”

可转化为：

1. 进入注册页
2. 完成注册
3. 登录成功
4. 创建空间/项目
5. 上传 PDF
6. 等待处理完成
7. 进入摘要结果页
8. 检测结果存在

---

## 5.3 环境执行系统（Environment Adapter）

## 功能

将 agent 的结构化动作转换成真实交互。

## 第一阶段执行器

**Playwright**

## 支持动作

- 页面跳转
- 点击按钮
- 输入文本
- 选择下拉框
- 上传文件
- 键盘操作
- 等待元素出现
- 读取页面文本
- 截图

## 后续可扩展

- Appium：移动端
- PyAutoGUI：桌面端
- Computer-use harness：视觉驱动桌面控制

---

## 5.4 页面观察系统（Observer）

## 功能

在每一步行动前收集环境事实。

## 可收集数据

- 当前 URL
- 页面标题
- DOM 摘要
- 可点击元素列表
- 表单字段
- 主按钮候选
- 错误提示
- 弹窗状态
- 截图
- 控制台错误
- 网络失败信息

## 输出

一个结构化 `ObservedPageState`

---

## 5.5 决策系统（Planner / Controller Agent）

## 功能

结合：

- persona
- task goal
- 实时页面状态
- RAG 检索上下文
- 最近执行历史

生成下一步动作。

## 输出格式

例如：

```markdown
{
"action": "click",
"target": "button[text='Upload']",
"reason": "当前任务要求上传文件，页面存在明确上传按钮"
}
```

---

## 5.6 验证系统（Validator）

## 功能

判断动作是否推进任务，是否已成功、失败、卡住或偏航。

## 核心判断

- 成功：是否满足成功条件
- 进展：是否更接近目标
- 失败：是否触发严重错误
- 死循环：是否重复出现相同行动模式
- 摩擦：是否出现迷失、反复、误解

---

## 5.7 摩擦分析系统（Friction Analyzer）

## 功能

从整个执行过程中识别 UX 问题和流程问题。

## 典型指标

- 总步数
- 重复点击次数
- 返回上一步次数
- 错误恢复次数
- 页面停留时间
- 是否多次误点
- 是否反复搜索同类入口
- 是否需要外部知识才能理解页面

## 输出

结构化摩擦报告：

- 高摩擦步骤
- 可疑文案
- 迷失节点
- 恢复困难节点
- 建议优化项

---

## 5.8 报告生成系统（Report Engine）

## 功能

将执行日志、摩擦分析、任务结果整理为人类可读报告。

## 报告内容

- Persona 信息
- 任务定义
- 执行路径
- 关键截图
- 成功/失败判定
- 摩擦摘要
- 根因分析
- 修复建议
- 与历史版本对比

## 12\. API 设计建议

## 12.1 任务相关

- `POST /projects`
- `POST /personas`
- `POST /tasks`
- `POST /runs/start`
- `GET /runs/{id}`
- `GET /runs/{id}/steps`
- `GET /runs/{id}/report`

## 12.2 运行状态相关

- `GET /runs/{id}/status`
- `GET /runs/{id}/events`
- `POST /runs/{id}/cancel`
- `POST /runs/{id}/retry`

## 12.3 知识库相关

- `POST /knowledge/import`
- `GET /knowledge/search`
- `POST /failure-cases/import`

## 12.4 对比分析相关

- `GET /reports/compare?run_a=...&run_b=...`
